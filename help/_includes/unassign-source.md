# Desasignar orígenes

>[!NOTE]
>
>Al anular la asignación de un origen, se borran todos los datos de cantidad. La reasignación de un origen puede causar problemas con cantidades vendibles, reservas y pedidos pendientes. Se recomienda procesar todos los pedidos antes de anular la asignación de orígenes o realizar otros cambios de origen. También puede utilizar nuevos comandos para detectar y actualizar reservas. Consulte [[!DNL Inventory Management] Referencia de CLI](../inventory-management/cli.md) para obtener más información.
